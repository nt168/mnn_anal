## 三、文档导航

### 如何使用预备阶段文档

#### 阅读顺序
1. **1号报告** - 了解模型选择基础（从160+大模型中选择子集）
2. **2号报告** - 分析性能指标合理性（pp、tg、pp+tg、执行时间四项指标）
3. **3号报告** - 查看长度约束（实验确定的提示词长度区间）
4. **4号报告** - 验证测试方法（llama.cpp固定模式可行性）
5. **5号报告** - 获取参数体系（最终测试参数配置）

#### 执行顺序
- 严格的依赖关系：后续报告需要前置报告的结论作为输入
- 每个报告目录都有独立的README说明具体内容和方法
- 脚本文件位于各报告的`测试脚本/`目录

## 四、技术要点

- **推理引擎**：MNN推理引擎验证测试
- **测试模式**：llama.cpp固定提示词模式、Prefill/Decode分离测试
- **测试对象**：大模型全生态（文本、多模态、语音等领域）
- **分析指标**：pp值、tg值、pp+tg混合值、执行时间
