前期测试结论：
1. 硅前测试客观限制 
	硅前应该将现实世界运行时间现在3到4天，即72-96小时内，映射到硅前仿真系统系统上，预计约100秒时间。
    - 尚需确认1（100秒实际相当于物理世界多少时间？需要精确的比例尺）
    - 尚需确认2（仿真机运行时间是否包括操作系统启动时间、程序加载时间等其他开销？）
    - 尚需确认2（仿真机是否能调整到与D3000m开发版硬件参数完全一致？）
2. 模型的选择，在硅前和硅后的测试中，应该分别考虑
    1. 硅后，我们需要综合考虑各个维度，选择可以在16g内存的D3000m上运行的模型5-10个，尽量覆盖更多的维度
    2. 硅前模型选择，考虑到我们需要在100秒运行时间内完成，我们可能只能运行3个测试实例，其中每个实例选择一个不同的模型，其中两个较小（1B以下）模型，一个中型模型（1到2B）
    - 尚需确认1（每个模型测试一个性能点可能不足，最好能一个实验测试两个点？）
3. 可变提示词和文件提示词的必要性（-vp -pf参数的选择）
	经过测试，可变提示词和文件提示词几乎不会对性能造成影响，因此可以选择使用固定提示词
    - 尚需确认1（文件提示词为词库特殊的某些多模态模型准备，使用其他不特殊的多模态模型是否可行？）
    - 尚需确认2（多模态模型和非多模态模型之间是否存在不一致的性能表现？不同的多模态模型之间是否存在不一致的性能表现？）
4. Benchmark模式的选择（-kv -pg参数的选择）
	经过代码研究与实际测试表明，应该选择kv=true的模式，false模式及pg参数均无需使用，原理另文列出。
5. Loadtime是否加入loadtime测试（-load参数的选择）
	如果每轮均加入测试loadtime，测试程序会反复加载多次来计算标准差，可能会引入更多的运行时间，需要在硅前权衡
	- 尚需确认1（是否需要修改代码增加一个无标准差的结果）
6. 重复测试次数的选择（-rep参数的选择）
	经过测试，重复3次似乎已经获得较小的CV
	- 尚需确认1（2、3、4、5次实验的典型CV具体是多少？）
7. 其他基本参数的选择（-c -t -mmp -dyo参数的选择）
  -c, --precision <n>                       (default: 2) | Note: (0:Normal(for cpu bakend, 'Nornal' is 'High'),1:High,2:Low)
  -t, --threads <n>                         (default: 4)
  -mmp, --mmap <0|1>                        (default: 0)
  -dyo, --dynamicOption <n>                 (default: 0) | Note: if set 8, trades higher memory usage for better decoding performance

8. 核心参数的选择（-p -n的选择）
	- 在硅后，每个模型均经过从粗到细的一个普遍测试过程
        - 初测：p、n分别从32到1024,按64步进
        - 精测：根据初测结果具体分析
	- 在硅前，根据硅后测试选定一个或两个性能点
