# AI模型性能基准测试分析

基于Qwen 0.6B和Hunyuan 0.5B模型在物理硬件和模拟硬件环境下的性能对比分析项目。

## 📋 项目概览

### 测试模型
- **Qwen 0.6B**: 60亿参数的语言模型
- **Hunyuan 0.5B**: 50亿参数的语言模型

### 测试环境
- **物理硬件**: 直接硬件测试
- **模拟硬件**: 硬件加速模拟测试

### 测试查询
1. 数学计算: "2+3=？"
2. 古诗词: "春眠不觉晓"下一句？
3. 地理知识: "中国首都是哪？"
4. 符号识别: "人民币符号？"
5. 科学知识: "光速是多少？"

## 🎯 核心发现

### ⚠️ 重要限制：数据高方差性问题

**关键发现**:
- **总体变异系数**: 15.3%（工程不可接受水平）
- **不可解释变异**: 32.3%的变异无法控制
- **稳定性问题**: 3/4测试组变异系数>10%

### 🤔 统计显著 vs 实际意义

**统计检验结果**:
- **模型效应**: Hunyuan 0.5B在解码速度上显著优于Qwen 0.6B (p=0.002)
- **环境效应**: 模拟硬件环境显著优于物理硬件环境 (p=0.018)

**但实际意义存疑**:
- 高变异 = 低预测性 = 低实用价值
- 测试结果重现性差，难以工程应用
- 需先解决稳定性问题再进行性能对比

### 📋 修正后的实用结论
> **虽然统计检验显示差异存在，但高达15.3%的变异系数表明数据极不稳定。在解决测试稳定性问题之前，这些发现不足以支持可靠的工程决策。稳定性比平均性能更重要。**

详细质疑分析见: [variance_analysis_review.md](variance_analysis_review.md)

## ⚠️ 数据质量说明

### 已识别问题
- **Token截断**: 20%数据(4/20)显示512 tokens，可能被输出限制截断
- **小样本**: 仅5个查询类型，统计功效有限
- **查询偏差**: 截断集中在Hunyuan模型的特定查询

### 处理策略
- 使用完整数据(16样本)进行统计分析
- 保守解释结果，强调局限性
- 建议更大规模的验证研究

## 📈 应用建议

### 选择Hunyuan 0.5B的场景
- 性能优先的应用
- 高频交互场景
- 响应时间敏感的应用

### 选择Qwen 0.6B的场景
- 输出稳定性要求高
- 资源受限环境
- 预测性要求严格的场景

### 部署环境
- 优先考虑模拟硬件环境
- 但需验证实际业务场景效果

## 🔬 技术细节

### 分析结果
- **总体均值**: 21.932 tokens/s
- **模型均值**: Qwen=20.314, Hunyuan=24.630
- **环境均值**: Physical=20.474, Simulated=23.391

### 统计规格
- **F统计量**: 模型=15.345, 环境=7.479
- **效应量**: η²(模型)=0.413, η²(环境)=0.201
- **显著性水平**: α=0.05
